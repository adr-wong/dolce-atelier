

1. Visión General del Producto
1.1 Descripción
Dolce Atelier es una plataforma de comercio electrónico (B2C) enfocada en la venta de pasteles artesanales.
El sistema permite:
    • Comprar pasteles predefinidos 
    • Solicitar pasteles personalizados mediante recetas 
    • Elegir entre entrega a domicilio o retiro en tienda 

1.2 Propuesta de Valor
    • Compra rápida y sencilla 
    • Productos artesanales de alta calidad 
    • Personalización mediante recetas del cliente 
    • Experiencia digital fluida de inicio a fin 

1.3 Modelo de Negocio
Modelo: B2C
Fuentes de ingreso:
    • Venta directa de productos del catálogo 
    • Cobro por elaboración de recetas personalizadas 

1.4 Objetivos
    • Lanzar MVP funcional 
    • Alcanzar tracción inicial (pedidos reales) 
    • Validar modelo de personalización 
    • Mantener alta conversión en checkout

5. Requerimientos Funcionales (Resumen)
    • Autenticación completa 
    • Catálogo visible con SSR 
    • Carrito funcional 
    • Checkout con Stripe 
    • Webhook para confirmar pagos 
    • Historial de pedidos 
    • Sistema de recetas personalizadas 
    • Panel administrativo 

6. Requerimientos No Funcionales
Rendimiento
    • Tiempo de carga < 2s 
Seguridad
    • Variables en entorno 
    • Validación de webhooks 
    • Protección de rutas 
Escalabilidad
    • Soporte para múltiples usuarios concurrentes 
Mantenibilidad
    • Código tipado (TypeScript) 
    • Validaciones (Zod) 

7. Lógica de Negocio
Estados del Pedido
    • PENDIENTE 
    • PAGADO 
    • PREPARANDO 
    • LISTO 
    • EN_CAMINO 
    • ENTREGADO 
    • CANCELADO 

Estados de Receta
    • PENDIENTE 
    • REVISANDO 
    • COTIZADA 
    • ACEPTADA 
    • RECHAZADA 

Reglas Clave
    • El carrito no reserva stock 
    • El pago se confirma solo vía webhook 
    • Los precios se guardan como snapshot 
    • El backend valida autenticación siempre

9. Flujo Principal
    1. Usuario navega catálogo 
    2. Agrega productos al carrito 
    3. Se autentica 
    4. Crea pedido 
    5. Paga en Stripe 
    6. Webhook confirma pago 
    7. Pedido actualizado 
    8. Usuario recibe confirmación
